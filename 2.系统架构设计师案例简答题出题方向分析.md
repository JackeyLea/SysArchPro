# 系统架构设计师案例真题简答题出题方向分析

## 一、整体分析概述

根据对《系统架构设计师案例真题分类解析.pdf》的深入分析，案例简答题主要涵盖以下技术领域：
- 系统设计与建模（UML、数据流图、E-R图等）
- 系统架构设计与评估（质量属性、架构风格、架构评估）
- 数据库系统与缓存设计（Redis、MySQL、缓存策略）
- Web应用设计（MVC、SSM、微服务、分布式架构）
- 嵌入式系统架构设计
- 系统安全
- 项目管理

---

## 二、各技术领域出题方向详细分析

### 【领域1】系统设计与建模 - 重要程度：⭐⭐⭐⭐⭐

#### 出题类型1：UML图分析与设计
**常见出题模式：**
- "根据业务描述，补充完善用例图/类图/序列图/状态图中的空白"
- "说明XX图与YY图的区别"
- "分析序列图中的消息类型"

**具体示例：**
1. 用例图填充：补充参与者、用例名称（2021年11月真题）
2. 类图关系判断：关联、聚合、组合、泛化关系识别（2016年11月真题）
3. 序列图分析：同步消息、异步消息、返回消息的区分（2024年5月真题）
4. 状态图设计：根据业务流程设计状态转换（2015年11月真题）

#### 出题类型2：数据流图(DFD)分析
**常见出题模式：**
- "补充完善数据流图中的加工/数据存储/外部实体"
- "说明数据流图的分层细化数据平衡原则"
- "分析数据流图中的错误类型（黑洞、奇迹、灰洞）"
- "比较数据流图与系统流程图的区别"

**具体示例：**
1. 0层DFD补充：填写E1~E4外部实体和P1~P4加工（2019年11月真题）
2. 数据流补充：说明需要补充哪些数据流（2019年11月真题）
3. DFD基本元素说明：外部实体、加工、数据存储、数据流（2014年11月真题）

#### 出题类型3：E-R图与数据建模
**常见出题模式：**
- "补充完善E-R图中的实体"
- "说明超类实体与派生属性的概念"
- "比较信息工程方法与面向对象方法"

**具体示例：**
1. E-R图实体填充：补充实体(1)~(6)（2022年11月真题）
2. 超类实体设计：抽象共同属性（2020年11月真题）

### 【领域2】系统架构设计与评估 - 重要程度：⭐⭐⭐⭐⭐

#### 出题类型1：质量属性效用树
**常见出题模式：**
- "填写质量属性效用树中的空白"
- "将需求描述归类到对应的质量属性"
- "说明质量属性场景6要素"

**具体示例：**
1. 效用树填充：性能、安全性、可用性、可修改性（2022年11月、2021年11月、2020年11月真题）
2. 质量属性识别：从12条需求中识别对应质量属性（2025年5月真题）
3. 质量属性场景：刺激源、刺激、环境、制品、响应、响应度量（2024年5月真题）

#### 出题类型2：架构风格比较与选择
**常见出题模式：**
- "比较两种架构风格的优缺点"
- "针对系统需求选择更适合的架构风格并说明原因"
- "从XX方面对比分析架构风格"

**具体示例：**
1. 解释器 vs 面向对象：折扣规则可修改性、灵活性、性能比较（2022年11月真题）
2. 管道过滤器 vs 仓库：数据处理方式、可扩展性、性能比较（2020年11月真题）
3. 解释器 vs 管道过滤器 vs 隐式调用：机器学习流程灵活性、算法扩展性（2021年11月真题）
4. 基于规则 vs 面向对象：用户级别、折扣规则灵活性、可扩展性（2019年11月真题）

#### 出题类型3：架构风险、敏感点、权衡点
**常见出题模式：**
- "给出架构风险、敏感点、权衡点的定义"
- "从需求描述中识别风险/敏感点/权衡点"

**具体示例：**
1. 定义与识别：从(a)~(o)中选出对应描述（2017年11月、2015年11月、2014年11月真题）

#### 出题类型4：微服务架构
**常见出题模式：**
- "简述微服务架构，对比单体架构优缺点"
- "说明微服务架构的特点"

**具体示例：**
1. 微服务vs单体：服务松耦合、独立扩展、技术多样性、运维要求（2024年5月真题）

### 【领域3】数据库系统与缓存设计 - 重要程度：⭐⭐⭐⭐⭐

#### 出题类型1：Redis缓存技术
**常见出题模式：**
- "说明Redis数据类型及其适用场景"
- "比较RDB与AOF持久化方式"
- "解决缓存穿透/雪崩/击穿的方案"
- "说明缓存一致性方案"

**具体示例：**
1. Redis数据类型选择：String、List、Set、Hash、ZSet应用场景（2020年11月真题）
2. RDB vs AOF：磁盘更新频率、数据安全、一致性、重启性能、文件大小（2020年11月真题）
3. 缓存穿透解决：布隆过滤器原理与优缺点（2022年11月真题）
4. 缓存雪崩解决：过期时间随机化、多级缓存、串行化更新（2019年11月真题）
5. 缓存一致性：旁路缓存模式、延迟双删、分布式锁（2024年11月真题）

#### 出题类型2：缓存分片与一致性哈希
**常见出题模式：**
- "说明哈希算法与一致性哈希算法的原理"
- "说明采用一致性哈希的原因"

**具体示例：**
1. 一致性哈希：节点增减只影响局部数据，迁移量小（2022年11月真题）

#### 出题类型3：数据库设计与反规范化
**常见出题模式：**
- "说明反规范化设计方法"
- "说明解决数据不一致性的方法"
- "选择合适的数据库类型"

**具体示例：**
1. 反规范化方法：增加冗余列、增加派生列、重新组表、分割表（2021年11月真题）
2. 数据一致性解决：定期同步、触发器、应用程序级同步（2021年11月真题）
3. 图数据库选择：知识图谱存储推荐Neo4j（2025年5月真题）

#### 出题类型4：Redis与MySQL数据同步
**常见出题模式：**
- "说明Redis与MySQL数据同步方案"
- "比较实时同步与异步准实时同步"

**具体示例：**
1. 同步方案：应用层双写、Binlog异步同步、消息队列、延迟双删（2021年11月真题）

## 三、常见出题模式总结

### 模式1：概念解释型
**特点：** 要求用200-300字说明某个技术概念
**高频考点：**
- 数据流图基本元素及作用
- 质量属性效用树
- 架构风险/敏感点/权衡点定义
- MVC架构风格
- Redis持久化方式
- 微服务架构特点

### 模式2：比较分析型
**特点：** 从多个维度比较两种技术/架构
**高频考点：**
- Lambda vs Kappa架构
- RDB vs AOF持久化
- TCP vs UDP协议
- 单体架构 vs 微服务架构
- 哈希算法 vs 一致性哈希

### 模式3：方案选择型
**特点：** 针对场景选择技术方案并说明原因
**高频考点：**
- 架构风格选择（解释器/管道过滤器/仓库）
- 缓存一致性方案选择
- 数据库类型选择
- 通信协议选择

### 模式4：问题解决方案型
**特点：** 分析系统问题并提出解决方案
**高频考点：**
- 缓存穿透/雪崩/击穿解决方案
- 高并发架构设计
- 数据一致性问题解决
- SQL注入/XSS攻击防护

### 模式5：架构图填充型
**特点：** 根据架构描述补充组件
**高频考点：**
- 质量属性效用树填充
- SSM/MVC架构图填充
- 大数据架构图填充
- 分布式缓存架构图填充

## 五、备考建议

1. **重点掌握质量属性**：性能、可用性、安全性、可修改性的定义、场景、实现方法
2. **熟练架构风格对比**：解释器、管道过滤器、仓库、面向对象、基于规则等
3. **深入理解Redis**：数据类型、持久化、缓存问题解决方案
4. **掌握UML图**：用例图、类图、序列图、状态图、数据流图
5. **关注新兴技术**：区块链、DDD、微服务、大数据架构
